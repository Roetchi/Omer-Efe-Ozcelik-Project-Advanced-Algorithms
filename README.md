# B208 Advanced Algorithms Project
## Potsdam hospital pathfinding with Dijkstra and A*

The code provided here is for my Individual B208 Project, which was based on creating a weighted graph of Potsdam, Germany, using Dijkstra and A*, in order to determine the shortest route from multiple starting points to the closest hospital.

## Included files
- `data/potsdam_graph.json` - curated Potsdam graph dataset
- `src/algorithms.py` - Dijkstra and A* implementations
- `src/graph_model.py` - graph loading and distance utilities
- `src/evaluate.py` - benchmarking, validation, and metrics collection
- `src/visualize.py` - figure generation
- `src/main.py` - full project runner
- `outputs/` - generated tables and figures

## How to run
```bash
pip install -r requirements.txt
python src/main.py
```

## Generated outputs
Running the project produces:
- `outputs/benchmark_results.csv`
- `outputs/benchmark_summary.csv`
- `outputs/graph_overview.png`
- `outputs/runtime_comparison.png`
- `outputs/node_expansions.png`

## Summary
My project will evaluate six different routing options across Potsdam and compare results of Dijkstra's and A*'s shortest paths versus those generated by an independent Oracle, Floyd-Warshall. Both Dijkstra's and A* produced shortest routes; however, A* produces fewer average searches, less memory usage and faster runtimes than Dijkstra's on the graphs I have created.
